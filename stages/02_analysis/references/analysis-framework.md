# Analysis Framework (Layer 3)

## Insight quality bar

An insight is NOT a summary of a finding. It must pass:

1. **So what?** — a busy reader immediately sees why it matters to their decision.
2. **Surprise or confirmation?** — state which. Confirming the client's prior is
   still useful, but say so.
3. **Falsifiable** — "what would change our mind" is answerable.

## Structure per insight

```
### Insight N: <one-sentence claim>
- Evidence: <cited findings [Sn]>
- Confidence: high | medium | low — <why>
- Changes if: <what new evidence would overturn this>
```

## Recommendations

- Max 5. If you have 8 recommendations you have none.
- Each: action, owner (who at the client), first step, main risk.
- Order by impact on the decision, not by topic.

## The counter-case

Steelman, not strawman: build the best opposing argument from the same findings.
If the recommendation survives it, say why. If it only survives by assumption,
label the assumption.
