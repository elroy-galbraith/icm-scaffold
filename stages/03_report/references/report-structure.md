# Report Structure (Layer 3)

## Section order

1. **Executive summary** — the answer, the top 3 insights, the recommendation. One page max.
2. **The question & scope** — from `client-brief.md`, two paragraphs.
3. **Insights** — one section per insight, evidence-first, confidence stated.
4. **Recommendations** — action, first step, owner, risk. Table format.
5. **Counter-case & limitations** — the steelman from stage 02, plus research gaps.
6. **Sources** — only sources cited in this report.

## Length discipline

If over the length target, cut in this order:
1. Background/context prose (the reader knows their own industry)
2. Redundant evidence (keep the strongest 2 citations per claim)
3. Lower-confidence insights (move to an appendix)

Never cut: the counter-case, confidence labels, or the sources section.

## Formatting

- Client-facing: no internal jargon, no stage/pipeline references, no metadata
  block visible in any exported version (the HTML comment is fine in markdown).
- Tables for recommendations and sources. Prose for insights.
