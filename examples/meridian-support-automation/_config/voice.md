# Voice Guide (Layer 3 — configure once per client)

How all prose in this workspace should sound. Stage contracts cite this file;
the agent internalizes it as constraints, not as content to process.

## Tone

- Professional but direct. Write like a trusted advisor, not a vendor.
- Confident where evidence supports it; explicit about uncertainty where it doesn't.
- No hype words: avoid "revolutionary", "game-changing", "cutting-edge", "leverage".

## Style

- Short sentences. Active voice. Concrete nouns.
- Lead with the conclusion, then support it (BLUF: bottom line up front).
- One idea per paragraph. Paragraphs under 5 sentences.
- Numbers over adjectives: "reduced cost 23%" not "significantly reduced cost".

## Audience

Meridian Outdoor Gear's COO: senior, time-poor, commercially literate, not technical.
Assume business/commercial literacy but zero AI/technical background — define any
AI or support-ops term listed in `shared/glossary.md` on first use. No filler.

## Never

- Never pad. If a section has nothing to say, cut the section.
- Never present speculation as fact; label assumptions explicitly.
- Never bury a recommendation below its justification.
